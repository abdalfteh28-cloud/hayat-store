# تفعيل متجر حياة للبيع — ما تحتاجه

تم ضبط المتجر ليكون جاهزاً لبيع المنتجات مع تسجيل بيانات العميل، اختيار الشحن، والدفع. هذا الملف يلخص ما تم وما تحتاج لتفعيله كمتجر حقيقي.

---

## ما تم ضبطه

### 1. صفحة تسجيل بيانات العميل (إتمام الطلب)
- **الخطوة ١ — بيانات العميل:** اسم، جوال، بريد، **عنوان تفصيلي** (المدينة، الحي، الشارع، رقم المبنى، الرمز البريدي)، مع أيقونات واضحة لكل حقل.
- **الخطوة ٢ — شركة الشحن:** اختيار طريقة التوصيل (عادي 3–6 أيام، سريع 2–4 أيام) أو إحدى شركات الشحن إذا تم تفعيلها من الإعدادات.
- **الخطوة ٣ — الملخص والدفع:** عرض المجموع والخصم والشحن ثم «تأكيد الطلب والدفع».
- بعد إتمام الطلب تظهر **صفحة شكر** تحتوي على رقم الطلب و«قيد المراجعة».

### 2. تسجيل الدخول للوحة التحكم
- إذا **لم** يُضبط «رابط إرسال الكود (OTP)» في الإعدادات، يمكن الدخول مباشرة بـ **اسم المستخدم** و**كلمة المرور** فقط (بدون كود جوال).
- بيانات الدخول: `HASSAN.1949` / `Aa@191610` — تم التأكد من أن التحقق يعمل معها بعد ضبط الـ trim للاسم وكلمة المرور.

### 3. الشحن
- **توصيل عادي:** 3–6 أيام (السعر من إعدادات المتجر).
- **توصيل سريع:** 2–4 أيام (السعر من إعدادات المتجر).
- إمكانية إظهار خيارات شركات الشحن (أرامكس، إم إس إم إس، DHL، ريد بوكس) من تبويب «بوابات الشحن» في الإعدادات.

### 4. الطلبات وحالاتها
- في لوحة التحكم → الطلبات يمكن **تغيير حالة الطلب** (قائمة منسدلة): بانتظار الدفع، مؤكد، مدفوع، تم الشحن، ملغي.
- عند تغيير الحالة إلى **مؤكد** أو **تم الشحن** يمكن إرسال إشعار للعميل إذا تم ضبط «رابط إشعار العميل».

## ما تحتاج لتفعيله ليكون المتجر جاهزاً للبيع

| المطلوب | الوصف |
|--------|--------|
| **1. بوابة الدفع (Moyasar)** | من إعدادات المتجر أدخل **مفتاح Moyasar السري** و**رابط الموقع (BASE_URL)**. بدونها الطلبات تُحفظ لكن لا ينتقل العميل لصفحة الدفع. |
| **2. رابط الموقع (BASE_URL)** | يجب أن يكون الرابط الحقيقي للمتجر (مثلاً `https://hayat-store.onrender.com`) ليعمل callback الدفع وإشعارات الطلبات بشكل صحيح. |
| **3. إشعار العميل بعد الطلب (رسالة شكر + رقم الطلب)** | بعد إتمام الطلب يظهر للعميل في نفس الصفحة رسالة «شكراً لك! تم استلام طلبك» ورقم الطلب و«قيد المراجعة». لإرسال **نفس الرسالة عبر واتساب أو SMS** تحتاج ربط خدمة إرسال (بوت واتساب أو بوابة SMS) مع **رابط إشعار العميل** (انظر أدناه). |
| **4. إشعار عند «تم قبول الطلب»** | عند تغيير حالة الطلب إلى **مؤكد** من لوحة التحكم، لإبلاغ العميل برسالة «تم قبول طلبك» تحتاج ضبط **رابط إشعار العميل** في الإعدادات. |
| **5. إشعار «تم الشحن»** | عند تغيير حالة الطلب إلى **تم الشحن**، لإبلاغ العميل تحتاج نفس **رابط إشعار العميل**. |

## رابط إشعار العميل (Customer Notify Webhook)

في **إعدادات المتجر** يوجد حقل: **«رابط إشعار العميل (عند قبول الطلب أو الشحن)»**.

- عند تغيير حالة الطلب إلى **مؤكد** أو **تم الشحن** يرسل الخادم طلب **POST** إلى هذا الرابط بالجسم التالي (JSON):

```json
{
  "orderId": 1234567890,
  "phone": "966501234567",
  "name": "اسم العميل",
  "status": "confirmed",
  "message": "تم قبول طلبك #1234567890"
}
```

أو عند الشحن:

```json
{
  "orderId": 1234567890,
  "phone": "966501234567",
  "name": "اسم العميل",
  "status": "shipped",
  "message": "تم شحن طلبك #1234567890"
}
```

- يمكنك إنشاء **خدمة صغيرة (مثلاً على Vercel أو سيرفرك)** تستقبل هذا الطلب وترسل رسالة واتساب أو SMS إلى `phone` مع نص `message`.
- أو ربط هذا الرابط ببوابة واتساب Business API أو أي مزود SMS يدعم webhook.

## ملخص سريع

- **صفحة بيانات العميل:** تعمل مع أيقونات وعنوان تفصيلي وثلاث خطوات (بيانات → شحن → دفع).
- **تسجيل الدخول للأدمن:** يعمل بـ HASSAN.1949 / Aa@191610 (ودون OTP إذا لم يُضبط رابط الكود).
- **عرض شركة الشحن:** يظهر للعميل في الخطوة الثانية مع التوصيل العادي 3–6 أيام والسريع 2–4 أيام.
- **بعد الطلب:** تظهر صفحة شكر ورقم الطلب و«قيد المراجعة».
- **إشعارات العميل (واتساب/SMS):** تحتاج ضبط **بوابة الدفع** و**رابط إشعار العميل** وربطها بخدمة إرسال الرسائل.

بعد ضبط بوابة الدفع ورابط الموقع يمكنك استقبال الطلبات والدفع. لإرسال رسائل تلقائية للعميل (شكر، قبول، شحن) تحتاج تفعيل «رابط إشعار العميل» وربطه بخدمتك.
