# رفع المشروع على Render — والرابط بعد النشر

تم تطبيق التعديلات التالية قبل الرفع:

- **حفظ بيانات العميل:** عند إتمام الطلب تُحفظ بيانات العميل (الاسم، الجوال، العنوان، البريد، الملاحظات والمنتجات) بشكل صحيح في الخادم.
- **عرض التطبيق مباشرة:** عند فتح الرابط يظهر المتجر فوراً (صفحة HTML) بدون ظهور أكواد أو JSON.
- **الاحتفاظ بالتعديلات والبيانات:** المنتجات (`data/products.json`) والإعدادات (`data/config.json`) مُضمّنة في المشروع ولا تُستثنى من Git، فستُرفع مع الكود ويتم الاحتفاظ بها عند كل نشر.

## خطوات الرفع على Render (أنت تنفّذها)

### 1) رفع التعديلات إلى GitHub

من **PowerShell** أو **CMD**:

```bash
cd "c:\Users\hardw\OneDrive\Documents\مجلد جديد\متجر حياة"
git add .
git commit -m "الاحتفاظ بالتعديلات وبيانات المنتج والإعدادات"
git push origin main
```

إذا كان الفرع عندك `master`:

```bash
git push origin master
```

أدخل اسم مستخدم GitHub وكلمة المرور (أو Token) عند الطلب.

### 2) النشر على Render

- إذا المشروع مربوط مسبقاً بـ Render، النشر يبدأ تلقائياً بعد `git push`.
- انتظر **2–5 دقائق** حتى يكتمل النشر.

### 3) فتح الرابط

- المتجر: **https://hayat-store.onrender.com**  
  يفترض أن تفتح الصفحة وتظهر واجهة المتجر مباشرة.
- لوحة التحكم: **https://hayat-store.onrender.com/admin**

### 4) إعدادات مهمة على Render

- **Environment:** أضف إذا تحتاج:
  - `NODE_VERSION` = `18`
  - `BASE_URL` = `https://hayat-store.onrender.com` (بدون `/` في النهاية)
  - `MOYASAR_SECRET_KEY` إذا تستخدم بوابة الدفع
- **Build Command:** `npm install`
- **Start Command:** `node server.js` أو `npm start`

| الطلبات الجديدة على السيرفر | — | على Render المجاني القرص مؤقت؛ عند إعادة النشر قد تُفقد الطلبات الجديدة إلا إذا فعّلت قرصاً ثابتاً من لوحة Render |
