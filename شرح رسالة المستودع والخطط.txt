  "معاينات PR والنشر التلقائي متاحة فقط للمستودعات
   المكوّنة مع المخططات"
═══════════════════════════════════════════════════════════════

معنى الرسالة:
  بعض المنصات (مثل Vercel أو Netlify أو غيرها) تتيح المعاينات
  والنشر التلقائي للمستودعات فقط عندما يكون المستودع على
  "خطة" معينة (مدفوعة أو فريق) أو مكوّن بإعدادات خاصة.


ما الذي يمكنك فعله؟
─────────────────────

  ► خيار ١ — استخدام Render (مجاني بدون اشتراك GitHub مدفوع)
    • Render لا يطلب "خطط" للمستودع.
    • سجّل في https://render.com
    • New → Web Service
    • وصّل مستودع GitHub (حتى المجاني)
    • Build Command: npm install
    • Start Command: npm start
    • اضغط Create. سيعطيك رابطاً مجانياً للمتجر.

  ► خيار ٢ — إن ظهرت الرسالة على Render نفسها
    • تأكد أنك تربط "مستودع GitHub" وليس فرعاً فقط.
    • يمكنك إنشاء Web Service يدوياً بدون استخدام render.yaml:
      نفس الأوامر أعلاه (npm install / npm start).

  ► خيار ٣ — منصة أخرى
    • Railway: railway.app — ربط المستودع ثم Deploy.
    • Koyeb: koyeb.com — نفس الفكرة، خطة مجانية.


ملف render.yaml في المشروع
  تمت إضافة ملف render.yaml لتهيئة المشروع لـ Render.
  يمكن لـ Render استخدامه تلقائياً عند الربط، أو يمكنك
  تجاهله وإدخال الأوامر يدوياً (npm install و npm start).


خلاصة
  الرسالة لا تعني أن النشر المجاني مستحيل. تعني أن تلك
  الميزة (معاينات PR أو نشر تلقائي) على تلك المنصة مرتبطة
  بخطة أو إعداد معيّن. استخدم Render أو Railway أو Koyeb
  وربط المستودع المجاني — يعمل النشر بدون "خطط" مدفوعة.
